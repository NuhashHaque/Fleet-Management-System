# Contributing

Thank you for your interest in contributing.

## Development setup

1. Fork the repository.
2. Create a feature branch from `main`.
3. Run the project locally with Docker Compose.
4. Ensure tests and linting pass before opening a pull request.

## Branch naming

Use one of:
- `feat/<short-description>`
- `fix/<short-description>`
- `chore/<short-description>`

## Commit messages

Use clear, imperative commit messages. Example:
- `feat(gateway): add telemetry validation endpoint`
- `fix(etl): prevent duplicate event inserts`

## Pull request checklist

- [ ] Scope is focused and minimal
- [ ] Documentation is updated where needed
- [ ] No secrets or credentials are committed
- [ ] Related issue is linked

## Code of conduct

By participating, you agree to follow the Code of Conduct in this repository.
